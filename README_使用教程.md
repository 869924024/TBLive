## 大文直播刷量客户端 - 工人使用教程（简明版）

> 本文给一线同学使用，配图你可自行按步骤截图补充。

### 一、启动前准备

- **接口地址**：例如 `http://服务器IP:5000`
- **客户端密钥（client_key）**：管理员分配

说明：程序启动会自动做一次过滤（12 小时冷却），顶部“可用 Cookie / 可用设备”显示的是过滤后的可用数量。

### 二、基础配置

1. 打开程序。
2. 在设置区填写：
   - 接口地址：粘贴服务器地址
   - 客户端密钥：粘贴你的 key
3. 点击“保存配置”。

提示：启动后 UI 会自动过滤冷却中的资源；Cookie 下拉只展示“可用”的账号。

### 三、手动拉取资源

- 拉取 Cookie：点击“拉取 Cookie”，每次只拉取 1 个。
- 拉取设备：点击“拉取设备”，每次固定拉取 200 个。
- 拉取完成后，程序会再次过滤，并更新“可用 Cookie / 可用设备”的数量。
- 资源使用完成会被自动标记冷却 12 小时，期间不会再次被拉取或选择。

异常退出说明：服务器会自动清理超过 1 小时的过期锁，如仍异常联系管理员处理。

### 四、选择参数并开始

1. 在“选择 Cookie”下拉中选择账号（只显示可用项）。
2. 设置参数：
   - 操作倍数：按任务要求填写
   - 使用设备数量：不超过“可用设备”数量
   - 代理：可不填；如走 API 模式，填入你的代理 API 地址
3. 点击“开始刷量”。

执行过程：

- 预热阶段：自动在可用设备中切换；
- 突发发送：异步并发；
- 任务完成后展示操作前/后观看数与增量。
- 增量监测固定 2 次：第 1 次 5 秒后，第 2 次 10 秒后。

### 五、规则与状态

- Cookie、设备成功使用后均进入 12 小时冷却，不会出现在下拉和可用计数中。
- 触发 “robot” 的 Cookie 也为 12 小时冷却（不永久封禁）。
- 顶部“可用 Cookie / 可用设备”为过滤后的真实可用数量。

### 六、常见问题（Q&A）

Q1：看得到 Cookie 但无法选择？

- 冷却中的 Cookie 不会出现在下拉；等待 12 小时或再次拉取。

Q2：为什么总是拉到同一个？

- 程序会在拉取后立即标记冷却，仍重复请让管理员检查服务器资源池。

Q3：任务完成但增量没变化？

- 等待两次监测（5s 与 10s）。仍无变化可能是接口延迟或风控限制。

### 七、建议截图点位

- 顶部“直播间信息”（直播间 ID + 获取观看数按钮）
- 中部统计条（可用 Cookie、可用设备、操作前/后观看数、增量）
- “选择 Cookie”下拉（仅显示可用）
- “操作倍数 / 使用设备数 / 代理设置”区域
- “开始刷量”按钮与“操作日志”（展示一次完整任务日志）
